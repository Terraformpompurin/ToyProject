🏗️ Terraform ISMS-P Scanner 시스템 아키텍처

각 컴포넌트에 마우스를 올리면 상세 설명을 볼 수 있습니다

① 입력 단계
1
👤 사용자
웹 브라우저를 통해
.tf 파일 / terragoat.zip 업로드
사용자가 취약한 테라폼 코드(.tf 또는 .zip)를 웹 프론트엔드를 통해 업로드합니다.
↓
2
⚡ FastAPI 백엔드
파일 수신 → 임시 디렉터리 저장
ZIP 압축 해제 → 스캔 준비
업로드된 파일을 서버의 임시 디렉터리에 안전하게 저장하고, ZIP이면 압축을 해제하여 스캔 준비를 마칩니다.
→
subprocess 실행
② 분석 단계
3
📋 커스텀 정책
checks/ 폴더
21개 ISMS-P 커스텀 룰
미리 정의된 21개의 ISMS-P 커스텀 보안 정책이 Checkov에 주입되어 정적 분석에 사용됩니다.
↓
4
🔍 Checkov 스캔 엔진
테라폼 코드 정적 분석
기본 정책 + 커스텀 정책 적용
백엔드가 subprocess로 Checkov를 실행합니다. 스캔 폴더 경로를 넘겨주면 내장 정책과 커스텀 정책으로 분석합니다.
→
원본 JSON 반환
③ 매핑 및 강화 단계
5
📚 ISMS-P 지식 베이스
isms_p_data.json
한국어 항목명 · 위험 설명 · 수정 가이드
서버 시작 시 메모리에 올려둔 JSON DB입니다. ISMS-P 번호로 조회하면 한국어 상세 정보를 반환합니다.
↓
6
📄 원본 JSON 결과
위반 리소스 + CKV ID 목록
(예: CKV_AWS_157)
Checkov가 반환한 raw JSON입니다. 어떤 리소스가 어떤 Check ID를 위반했는지 담겨 있습니다.
↓
7
🗺️ 룰 매핑 로직
RULE_MAPPING 딕셔너리 조회
CKV_AWS_157 → ISMS-P 2.6.1
CKV ID를 ISMS-P 항목 번호로 변환하는 딕셔너리입니다. 매핑 후 지식 베이스에서 한국어 정보를 가져옵니다.
↓
8
✅ 가공된 결과
한국어 가이드 포함
results 리스트 완성
영어 에러 메시지에 ISMS-P 정보와 한국어 가이드를 합쳐 프론트엔드가 바로 사용할 수 있는 데이터로 만들어집니다.
→
최종 JSON 전달
④ 시각화 단계
9
📦 최종 JSON 응답
가공 완료된 데이터
프론트엔드로 전송
백엔드가 가공이 완료된 최종 JSON을 프론트엔드로 반환합니다.
↓
10
⚛️ React 프론트엔드
ViolationTable 렌더링
요약 카드 · 파이 차트 시각화
React가 JSON 데이터를 받아 ViolationTable, 요약 카드, 파이 차트 등을 실시간으로 업데이트하여 사용자에게 보여줍니다.
